# 用例生成共通原则

功能测试用例与接口测试用例 **共享** 以下约束；子 Skill 在此基础上追加专属规则。

## 材料依据（禁止编造）

| 类型 | 事实来源 | 读不到时 |
|------|----------|----------|
| 功能用例 | PRD / 需求文档 / 用户故事 | 标 **待确认-产品**，不得写死必过预期 |
| 接口用例 | OpenAPI 契约（`6.测试用例生成/docs/contracts/`） | 标 **待确认-开发**，不得虚构 path/状态码/字段 |

**未读取到材料时：停止生成必过用例**，仅可输出待确认说明与所需文件路径。

## 待确认标注格式

不明确或文档未定义的内容，统一写作：

```text
待确认-{角色}：{一句说明缺什么、问谁}
```

| 角色 | 典型场景 |
|------|----------|
| 产品 | 业务规则、验收边界、状态机含义 |
| 开发 | 错误码触发条件、未文档化行为、鉴权细节 |

## PRD / 契约不清晰（写用例 vs 执行后优化）

| 阶段 | 做法 |
|------|------|
| **写用例 / 骨架** | **待确认-{角色}**；禁止编造必过预期；**禁止预设** xfail/skip |
| **pytest 执行后** | 在 **二次 check、失败修复闭环、一页简报** 中对比 **实际结果** 决策 xfail/skip |

执行后 xfail/skip 判定与 reason（须含 trace 证据）→ [接口自动化实现/xfail与skip治理.md](../接口自动化实现/xfail与skip治理.md)。

## 脱敏（对话与用例表）

| 禁止出现 | 替换方式 |
|----------|----------|
| 真实 IP、端口、内网 URL | `[ENV_API_BASE_URL]` + path |
| 明文 password / token / 密钥 | `[ENV_DEMO_PASSWORD]`、`[ENV_JWT_TOKEN]` 等 |
| 真实手机号 / 身份证 | 见 [测试数据生成/脱敏规范.md](../测试数据生成/脱敏规范.md) |

用例表、简报、Agent 回复必须脱敏；**执行用数据文件**（`接口数据集/`、`数据集/valid/`）可保留完整虚构值。

## 输出质量门禁

1. 每条预期能指向材料依据（PRD 章节 / 契约 operation+schema）。
2. 接口用例预期遵循 **断言基本要求**：**先 body、再 status 联合断言**；**状态码 → body 信息 → body 字段** 对齐 API 文档；反向用例 **以绿色为目标** 设计。
3. 不把「待定」「TBD」写死为确定结论。
4. 生成后自检：无臆造、无敏感明文、编号与模板列齐全。
5. 无法断言的条目：写用例阶段标 **待确认**；执行后按 **四种修复方式** 归属（见 [用例修复四种方式.md](../接口自动化实现/用例修复四种方式.md)）。
