# 接口测试数据目录约定

接口用例 **f. 测试数据单独管理**：用例表只引用路径，payload 落独立文件。

## 目录结构

```text
工作区根目录/
├── 接口用例集/{domain}/
│   └── cases.md                 # 用例主表（含 path/method/headers/body 引用）
└── 接口数据集/{domain}/
    ├── POST_user_services_login_001.yaml
    ├── POST_user_services_login_002.yaml
    └── ...
```

`{domain}` 与契约子目录一致，如 `blackcore-mall`。

## 文件命名

与 **用例编号** 相同：`{用例编号}.yaml`（默认 yaml）。

## 单文件内容（示例）

```yaml
# POST_user_services_login_001 — 正向登录
headers:
  Content-Type: application/json
body:
  username: demo_user01
  password: "[ENV_DEMO_PASSWORD]"
notes: 执行时将 ENV_DEMO_PASSWORD 替换为环境变量或本地 .env（勿提交）
```

## 敏感数据规则

| 场景 | 规则 |
|------|------|
| 用例表 `headers` / `body` 列 | 仅写 `@接口数据集/...` 或占位符，**不写明文密码/Token** |
| 用例表「预期结果」 | 不写真实 response body 中的密钥 |
| `接口数据集/*.yaml` | 可用完整 **虚构** 值供脚本执行；password 仍建议 `[ENV_*]` 占位 |
| 对话 / 简报 | 按 [脱敏规范.md](../../测试数据生成/脱敏规范.md) 脱敏展示 |

## 与第五节课数据的关系

- `第五节课/数据集/`：Pydantic 业务实体（order_flow 等），走 **测试数据生成** Skill。
- `接口数据集/`（工作区根目录）：HTTP 请求级 payload，走 **api 子 Skill**。
- 同一业务流程可先有用例数据，再在接口 data 文件中 **引用或复制** 虚构值，勿混目录。

## 反向用例数据

每条反向用例 **独立文件**，禁止与正向共用同一 yaml（课内 S4：neg 独立文件）。
