# 功能用例二次 Check 简报模板

脚本 `func_case_second_check.py` 按此结构输出 Markdown（Check 通过后内嵌 **§9 质量评分**）。

## 文件名

```text
功能用例集/{主题}/second_check_brief_功能用例_{YYYYMMDD_HHMM}.md
```

文件名与 §1「评审时间」均精确到 **分钟**（不含秒）。

## 章节

| 章节 | 内容 |
|------|------|
| §1 基本信息 | 评审时间、主题、用例路径、Check 轮次、用例/模块/P0 统计、**质量总分/档位** |
| §2 总体结论 | **Check：通过/不通过**；**质量评估：XX 分（档位）**（Check 不过则「未评分」） |
| §3 覆盖核对 | 用例总数、模块数、正反向统计 |
| §4 分项 Check | F1～F12 表格 |
| §5 修复记录 | 各轮自动修复动作 |
| §6 遗留问题 | 仅 Check **不通过** |
| §7 提示项 | warn 级 |
| §8 后续建议 | Check 与评分分工、85+ 交付目标 |
| §9 质量评分 | F1～F8 八维表 + 总分 + 档位（Check 不过则「未评分」） |
| §10 改进建议 | 总分 **<85** 时列出质量提升项 |

## §2 结论示例

**通过 + 评分：**

> Check：**通过**  
> 质量评估：**88 分（良好）**

**不通过：**

> Check：**不通过**  
> 质量评估：**未评分**

## §9 评分表示例

| 代号 | 维度 | 得分 | 满分 | 得分率 | 说明 |
|------|------|------|------|--------|------|
| F3 | 正反向结构 | 12.0 | 15 | 80% | 3 条反向预期文案重复 |
| F5 | 预期可验证性 | 11.5 | 15 | 77% | 待确认预期 18 条 |

**合计：82 / 100（合格）**
