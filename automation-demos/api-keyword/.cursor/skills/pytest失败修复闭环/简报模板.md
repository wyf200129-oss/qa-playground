# pytest 执行与修复简报模板

执行 pytest 并完成（或终止）修复闭环后，**必须**落盘 Markdown 简报。

## 文件名

```text
第九节课/报告集/{scope}_repair_brief_{YYYYMMDD_HHMMSS}.md
```

- `{scope}`：被测目录简称，如 `demo_cases`、`tests`
- 时间精确到 **秒**，与 §1 执行时间一致

## 章节结构

| 章节 | 必填 | 内容 |
|------|------|------|
| §1 执行概要 | ✅ | 用例文件集、执行时间、pytest 命令、修复轮次上限/实际消耗 |
| §2 结果统计 | ✅ | total / passed / failed / error / skipped |
| §3 总体结论 | ✅ | **通过 / 不通过** + 一句话原因 |
| §4 修复轮次摘要 | 有修复时 | 各轮改了什么、回归命令与结果 |
| §5 人改 vs 机改 | ✅ | 条目或占比（诚实填写，无则写「无人工改代码」） |
| §6 回归范围 | ✅ | 实际目录 / mark / 单文件 |
| §7 用例明细 | ✅ | 每条用例 a～f（见下） |
| §8 残余风险 | ✅ | 全局级，勿写「已彻底解决」；**接口自动化**须交叉引用二次 check 简报 **§7 假绿**（见下） |
| §9 后续建议 | 可选 | 待人工项、文档补充、产品确认 |
| §10 执行后优化 | **四方式汇总** | ①红→绿 {A} ②维持红 {B} ③xfail {C} ④skip {D}（见 [用例修复四种方式.md](../../接口自动化实现/用例修复四种方式.md)） |

## 用例明细（§7）每条须含

| 字段 | 说明 |
|------|------|
| a. 初始失败现象 | 无失败写「执行即通过」；有失败附 **脱敏** trace 摘要（异常类型、文件:行、关键断言） |
| b. 根因类别 | `断言` / `数据` / `环境` / `产品缺陷` / `需求不明确` / `未触发失败`；不确定须标注 |
| c. 修改点 | 文件 + 函数/行；未改写「无」 |
| d. 人改 vs 机改 | 该条是否人工介入；默认机改标注 Agent/AI |
| e. 回归范围 | 该条修复后实际执行的 pytest 范围 |
| f. 残余风险 | 该条仍存疑或依赖未验证点 |

## §3 结论示例

**不通过：**

> **结论：不通过**  
> 原因：15 条用例中 3 条仍失败（2 条需求不明确、1 条产品缺陷待三轮升级），未达全绿。

**通过（含假绿提醒）：**

> **结论：通过**  
> 原因：目标范围内用例全部执行通过。  
> **假绿提醒**：{M} 条 status-only（见二次 check 简报 §7 / 本简报 §8）；pytest 绿 ≠ body 已验真。

## §8 假绿残余风险（接口自动化 scope 必填）

当被测范围为 **接口自动化 flows/scenarios** 且存在 **status-only** 断言时，§8 **必须**包含：

```text
假绿风险：{无 / 低 / 中 / 高} — status-only {M} 条（详见 second_check_brief §6/§7）
说明：上述用例 pytest 通过仅表示 status 符合预期，不保证 body 业务正确。
解决手段：{引用二次 check 简报 §7 已采纳项，或列出待办}
二次 check 简报：{path/to/second_check_brief_*.md}
```

无 status-only 时写「假绿风险：无（全部为 body+status）」。

## §10 用例修复四方式汇总（一页简报核心）

> 总纲：[用例修复四种方式.md](../../接口自动化实现/用例修复四种方式.md)

```text
① 红→绿：{A} 条（§4 修复轮次 / §7 明细）
② 维持红：{B} 条（§3 不通过 / §7 需求不明确 / 升级人工）
③ xfail：{C} 条（§10.1 明细，须执行证据）
④ skip：{D} 条（§10.1 明细，须执行证据）
```

### §10.1 ③ xfail / ④ skip 明细

| 用例 | 标记 | 环节 | 材料 vs 实际 | 证据 | 对接人 |
|------|------|------|--------------|------|--------|
| test_wrong_pwd | xfail | 修复闭环 | 契约 200/400 vs 实 401 | trace 2026-06-06 | 开发 |

### 一页小结摘要（§3 可引用）

```text
四方式汇总：红→绿 {A} | 维持红 {B} | xfail {C} | skip {D}
二次 check 交叉引用：{second_check_brief §8 路径，若适用}
```

## 脱敏要求

- 去掉 token、内网 host、真实账号路径
- 伪代码 demo 可直接引用 `PseudoResponse` 等占位结构

## 与 Skill 的关系

- 需求不明确 → loop 已终止的用例，§7 须写清 **为何不修**
- 三轮仍失败 → 引用 [升级人工模板](SKILL.md#升级人工模板)
- 简报生成是闭环 **最后一步**，与 `第九节课/第09节-pytest失败修复闭环与一页小结.md` 课堂模板对齐
