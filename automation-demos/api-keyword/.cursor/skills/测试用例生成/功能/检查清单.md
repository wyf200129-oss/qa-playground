# 功能用例二次 Check 清单

生成功能用例后 **必须** 执行本清单（由 `6.测试用例生成/scripts/func_case_second_check.py` 自动校验，最多 **2 轮**，可自动修复）。

## 检查项

| 代号 | 维度 | 检查内容 | 不通过时的自动修复 |
|------|------|----------|-------------------|
| **F1** | 格式 | 各表含核心列：用例编号、模块、用例标题、前置条件、测试步骤、预期结果、优先级、断言来源 | 无（需重新生成） |
| **F2** | 格式 | 编号 `FUNC-{模块缩写}-{三位序号}`（可选 `-R` 表反向） | 无（需人工改） |
| **F3** | 追溯 | 文首含需求来源（PRD / SRS 路径） | 无（warn） |
| **F4** | 语义 | 步骤/预期 **无** HTTP path、method、status、headers 等接口细节 | 无（需人工改） |
| **F5** | 脱敏 | 无 IP、完整 URL、明文口令、真实手机号 | 替换为占位符 |
| **F6** | 可执行 | 测试步骤以 `1.` 编号（多数用例） | 无（warn） |
| **F7** | 断言 | 预期结果非空；断言来源基本齐全 | 无 |
| **F8** | 覆盖 | 若有 `scope-coverage-matrix.md`，FP 功能点在用例表有对应 | 无（warn） |
| **F9** | 正反向 | 若有「用例类型」列，反向须关联有效正向编号 | 无（warn） |
| **F10** | 优先级 | 至少存在 P0 用例 | 无 |
| **F11** | 待确认 | 待确认格式 `待确认-{角色}：`；统计条数 | 统一冒号格式 |
| **F12** | 无编造 | P0 预期不得为空/占位且未标待确认 | 无 |

## 执行流程

```text
生成用例 → 第 1 轮 Check → 有问题则自动修复（脱敏/待确认格式）
         → 第 2 轮 Check → 仍有问题则 FAIL + 失败简报
         → 通过则 PASS + 成功简报 + 质量评分
```

## 命令

```bash
cd 6.测试用例生成
python scripts/func_case_second_check.py --topic 库存管理 --max-rounds 2
python scripts/func_case_second_check.py --topic 课程中心 --max-rounds 2
```

单独评分（须 Check 已通过）：

```bash
python scripts/func_case_quality_score.py --topic 库存管理
```
