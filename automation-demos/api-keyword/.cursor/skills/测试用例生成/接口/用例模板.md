# 接口用例表模板

## 固定表头（8 列）

生成接口用例时 **必须** 使用以下列（顺序不变）：

| 列名 | 说明 |
|------|------|
| **用例编号** | 见编号规范 |
| **path** | 契约中的路径，如 `/user_services/login`；**不含** 基址 |
| **method** | GET / POST / PUT / DELETE 等 |
| **headers** | 请求头；Token 用占位符；可写 `见数据文件` |
| **body** | 请求体摘要或 `@接口数据集/...` 引用；GET 可写 `-` |
| **预期结果（预期response）** | HTTP 状态码 + 契约内 response 要点 |
| **实际结果** | 执行后填写；生成阶段 **留空** 或 `(待执行)` |

## 用例编号规范

每个 operation **4 条**：1 正向 + 3 反向。

```text
{METHOD}_{path转下划线}_{001|002|003|004}
```

| 后缀 | 类型 | 说明 |
|------|------|------|
| `_001` | 正向 | 合法入参，期望契约成功码 |
| `_002` | 反向 1 | 鉴权/Token 类（缺 header、无效 token） |
| `_003` | 反向 2 | 参数类（缺 required、类型错误、越界） |
| `_004` | 反向 3 | 业务/资源类（404、余额不足等，**须契约有对应 response**） |

**path 转下划线示例**：`/user_services/login` → `user_services_login`

完整编号示例：`POST_user_services_login_001`

## Markdown 表示例

```markdown
# 黑核商城接口用例（契约 version: 0.0.1）

> 基址：`[ENV_API_BASE_URL]`（勿写入下表 path 列）

| 用例编号 | path | method | headers | body | 预期结果（预期response） | 实际结果 |
|----------|------|--------|---------|------|--------------------------|----------|
| POST_user_services_login_001 | /user_services/login | POST | Content-Type: application/json | @接口数据集/blackcore-mall/POST_user_services_login_001.yaml | 200；响应含 token 字段（契约 examples） | |
| POST_user_services_login_002 | /user_services/login | POST | Content-Type: application/json（无 Authorization） | @接口数据集/blackcore-mall/POST_user_services_login_002.yaml | 401；Invalid credentials | |
```

## 预期结果写法

**断言基本要求**：获取响应后 **先 body、再 status 联合断言**；预期须覆盖 **状态码 → body 信息 → body 关键字段**，均对齐 API 文档。

```text
{HTTP状态码}；{body 信息/错误语义}；{契约 responses 中 schema 或 description 要点}
```

示例：

```text
200；响应含 token 字段（契约 examples）
401；Invalid credentials；响应 body 不含可复用 token
422；缺少必填字段 username；与契约 422 description 一致
```

- 状态码 **必须** 在该 operation 的 `responses` 键中存在。
- **反向用例** 同样须写 body 错误要点 + status，且 **以可稳定通过（绿色）为目标** 设计。
- 不得断言契约未声明的字段为必过（除非标 `待确认-开发`）。
