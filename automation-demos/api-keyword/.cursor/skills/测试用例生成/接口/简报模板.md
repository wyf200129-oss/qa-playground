# 接口用例二次 Check 简报模板

脚本 `api_case_second_check.py` 按此结构输出 Markdown（Check 通过后内嵌 **§9 质量评分**）。

## 文件名

```text
接口用例简报集/{domain}_check_brief_{YYYYMMDD_HHMM}.md
```

文件名与 §1「评审时间」均精确到 **分钟**（不含秒）。

## 章节

| 章节 | 内容 |
|------|------|
| §1 基本信息 | 评审时间、domain、契约/version、用例与数据路径、Check 轮次、**质量总分/档位** |
| §2 总体结论 | **Check：通过/不通过**；**质量评估：XX 分（档位）**（Check 不过则「未评分」） |
| §3 覆盖核对 | operation 数、用例总数、正向/反向统计 |
| §4 分项 Check | C1～C14 表格 |
| §5 修复记录 | 各轮自动修复动作 |
| §6 遗留问题 | 仅 Check **不通过** |
| §7 提示项 | warn 级 |
| §8 后续建议 | Check 与评分分工、85+ 交付目标 |
| §9 质量评分 | S1～S8 八维表 + 总分 + 档位（Check 不过则「未评分」） |
| §10 改进建议 | 总分 **<85** 时列出质量提升项 |

## §2 结论示例

**通过 + 评分：**

> Check：**通过**  
> 质量评估：**93 分（优秀）**

**不通过：**

> Check：**不通过**  
> 质量评估：**未评分**

## §9 评分表示例

| 代号 | 维度 | 得分 | 满分 | 得分率 | 说明 |
|------|------|------|------|--------|------|
| S3 | 正反向结构 | 10.0 | 15 | 67% | 13 个接口反向预期文案重复 |
| S5 | 响应断言质量 | 14.2 | 15 | 94% | 待确认 7 条 |

**合计：93 / 100（优秀）**
