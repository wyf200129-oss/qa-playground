# 接口用例二次 Check 清单

生成接口用例后 **必须** 执行本清单（由 `6.测试用例生成/scripts/api_case_second_check.py` 自动校验，最多 **2 轮**，可自动修复）。

## 检查项

| 代号 | 维度 | 检查内容 | 不通过时的自动修复 |
|------|------|----------|-------------------|
| **C1** | 格式 | 8 列表头齐全且顺序正确 | 无（需重新生成） |
| **C3** | 格式 | 文首含契约路径与 `info.version` | 补全文首块 |
| **C4** | 脱敏 | path/headers/body/预期 无 IP、端口、完整 URL | 替换为 `[ENV_API_BASE_URL]` 等 |
| **C5** | 脱敏 | 无 `password123` 等明文演示口令 | 替换为 `[ENV_DEMO_PASSWORD]` |
| **C6** | 覆盖 | 契约每个 operation 均有对应用例 | 重新执行 `generate_api_cases.py` |
| **C7** | 数量 | 用例总数 = operation 数 × 4 | 重新生成 |
| **C8** | 正反向 | 每 operation 恰好 `_001`～`_004`（1 正 3 反） | 重新生成 |
| **C9** | 契约 | path + method 与 OpenAPI 一致 | 无（需人工改） |
| **C10** | 契约 | 非「待确认」预期的 HTTP 码 ∈ 该 operation `responses` | 改为 `待确认-开发：…` |
| **C11** | 数据 | body 列为 `@接口数据集/{domain}/{用例编号}.yaml` | 修正引用路径 |
| **C12** | 数据 | 每条用例有独立 yaml；case_id 一致 | 创建缺失占位文件 |
| **C13** | 格式 | 「实际结果」列留空 | 清空该列 |
| **C14** | 待确认 | 待确认项格式 `待确认-{角色}：`；统计条数 | 仅提示，不阻断 |

## 执行流程

```text
生成用例 → 第 1 轮 Check → 有问题则自动修复
         → 第 2 轮 Check → 仍有问题则 FAIL + 失败简报
         → 通过则 PASS + 成功简报
```

## 命令

```bash
cd 6.测试用例生成
python scripts/api_case_second_check.py --domain blackcore-mall --max-rounds 2
```

生成用例脚本会在末尾 **自动调用** 二次 Check（`generate_api_cases.py`）。
